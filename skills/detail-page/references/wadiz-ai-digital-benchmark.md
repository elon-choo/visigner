## What "Wadiz-grade" means here

These pages are not quiet SaaS landing pages. They are long-scroll sales documents where the reader needs repeated proof, not subtlety. The design often looks louder than a Western landing page, but the hierarchy is disciplined:
- One impossible-sounding result is repeated in many forms.
- Every 1-2 viewport heights contains a hard claim, proof object, price anchor, or deliverable.
- Large image bands do most of the work; DOM text is secondary.
- The right reward sidebar is always in the reader's peripheral vision on desktop, so the story body must justify the visible prices.
- The page constantly turns abstract digital products into concrete objects: book mockups, Figma screens, spreadsheet screenshots, VOD boxes, icon packs, open-chat access, challenge calendars.

## Shared conversion pattern

1. **Campaign thumbnail as punchline**
   - Big short claim: `AI 직원 10명`, `30분이면 자동 수익 세팅`, `외주비 0원`.
   - Asset collage: product mockups, tool logos, medal/badge, model or founder.
   - Bottom strip: event/deadline/freebie.

2. **Wadiz summary and right reward stack**
   - First screen pairs the story body with the sticky reward list.
   - The generated page must design both: body persuasion and reward pricing cannot be separate afterthoughts.

3. **Authority pile-up**
   - Funding amount, percent achieved, supporter count.
   - Category rank: `와디즈 1위`, `슈퍼메이커`, `누적 n억`.
   - Previous funding screenshots with yellow/gold highlight boxes.
   - Review captures, satisfaction score, follower count, media/award/company logos.

4. **Problem contrast**
   - Show the bad current state visually: weak product photo, ugly layout, confusing page, slow manual work.
   - Use blunt marks: red X, arrows, handwritten notes, black speech bubble.
   - Then present the mechanism: `디자인 X AI`, `AI봇`, `자동화 공식`, `프롬프트 세팅`.

5. **Mechanism before offer**
   - A strong mid-page section explains why the result is possible.
   - Visual forms: formula card, step ladder, process flow, prompt screen, AI tool stack, before/after comparison.
   - Avoid saying "our product helps"; show the mechanism as a reusable machine.

6. **Deliverable proof**
   - Digital goods must look physical.
   - Show PDF covers, VOD modules, Figma frames, spreadsheet/prompt libraries, templates, icon packs, bonus files.
   - Blur or crop enough to imply value without giving away the product.

7. **Curriculum and package density**
   - Long chapter lists justify price.
   - Package cards repeat: name, badge, checklist, bonus, original value, discount, final price.
   - The body repeats package economics even though Wadiz also has reward cards on the right.

8. **Urgency loops**
   - Opening strip, mid-page benefit block, package block, final event block.
   - Wadiz language: `72시간`, `얼리버드`, `울트라 얼리버드`, `오늘 23:59`, `한정`, `사전 알림`.

## Visual modules to generate

For an AI/digital Wadiz page, include at least 8 of these 12 modules:

| Module | Job | Visual recipe |
|---|---|---|
| Hero proof thumbnail | Stop scroll | Huge outcome claim + mockups + tool logos + medal/badge + event strip |
| WAi-style summary | Compress offer | 3 bullets: what it is / outcome / Wadiz-only benefit |
| Previous-funding proof | Borrow trust | Screenshot-like card + yellow/gold outline around funding numbers |
| Bad example contrast | Create pain | Before image, red X/arrows/handwriting, short "이래서 안 팔립니다" style copy |
| Mechanism reveal | Make the result believable | Formula/flow/step ladder with 3-5 steps and one highlighted verb |
| Tool stack | Make AI tangible | ChatGPT/Claude/Gemini/Midjourney/Figma-like tiles, not emoji bullets |
| Live demo screen | Show process | Prompt/chat/sheet/template screen inside a real device/browser frame |
| Review wall | Social proof | Realistic review cards, highlighted sentences, rating/stat block |
| Deliverables wall | De-risk digital goods | PDF/VOD/Figma/spreadsheet/icon-pack thumbnails in a dense grid |
| Challenge/calendar | Make execution concrete | Day 1-30 ladder, mission cards, progress motif |
| Package economics | Justify price | Original value crossed out, discount badge, checklist, recommendation medal |
| Event CTA band | Convert now | Dark/high-contrast band, one benefit, one deadline, one action |

## Page rhythm

Repeat this rhythm down the scroll:
1. **Claim band** — one huge sentence, 6-14 Korean words.
2. **Evidence object** — screenshot, mockup, review, graph, table, or demo.
3. **Interpretation line** — explain what the reader should conclude.
4. **Next curiosity hook** — a question or arrow to the next block.

If three consecutive screens are only text or only decorative visuals, the page is below benchmark.

## Visual identity guidance

Two proven directions from the captures:
- **Dark automation premium:** charcoal/black base, acid-lime or yellow focal text, white type, 3D/product mockups, console/process cards. Best for automation systems, monetization, productivity, high-ticket education.
- **Bright template/operator kit:** purple/blue base with mint and gold proof accents, white cards, medal/badge motifs, tool screenshots, Figma/template artifacts. Best for templates, design kits, AI workflow files.

Purple/blue is allowed for this category only when it is not the generic AI gradient default. It must be paired with concrete proof modules, a second accent family (mint/gold/red/black), and strong product artifacts. If the page could be mistaken for a generic AI SaaS hero, it fails.

## Generation checklist

- [ ] One message can be stated in one line: `AI로 [cost/time/pain]을 [specific result]로 바꾼다`.
- [ ] First 2 viewports contain: hero claim, product artifact, authority number, Wadiz-only benefit, visible reward logic.
- [ ] At least 8 benchmark modules are present.
- [ ] Every major claim has an evidence object in the next 1 viewport.
- [ ] Digital deliverables are shown as concrete artifacts, not just listed.
- [ ] There is a bad-example / before-after contrast block.
- [ ] There is a mechanism block before the package/price block.
- [ ] Package economics repeat in the story body, not only in the Wadiz sidebar.
