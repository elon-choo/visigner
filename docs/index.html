Left alone, a model samples the average of its training data: Inter, a purple gradient, four identical cards, all centered. That average is the AI-slop look. Visigner doesn't make the model smarter; it runs the same spine through every skill, rejects that average, and verifies from pixels, not vibes.

Visigner가 하지 않는 것What Visigner does NOT do

위 수치를 믿어도 되는 이유는, 같은 잣대로 "없는 것"까지 그대로 공개하기 때문입니다. The inventory above is trustworthy because the same yardstick discloses what's absent:

  • 벤치마크 수치를 만들지 않습니다. 전환율·속도 같은 외부 성능 벤치마크가 저장소에 없습니다. 마케팅할 수 있는 건 인벤토리 카운트(스킬 8·에이전트 3·커맨드 11·스크립트 30·텔 검출기 22)와 두 게이트(10차원 안티-슬롭 루브릭 + 취향 심사단)·WCAG 2.2 AA 목표뿐입니다.It invents no benchmark numbers. There's no external performance benchmark (conversion, speed) in the repo. What's marketable is inventory counts (8 skills · 3 agents · 11 commands · 30 scripts · 22 tell detectors) and the dual gate (10-dim anti-slop rubric + a taste jury) · WCAG 2.2 AA target.
  • 검사 규칙은 코드로 있고, CI가 강제합니다. 텔 검출기 22종이 scripts/anti-ai-eval.js에 있고, 회귀 테스트 5개 파일이 매 푸시마다 GitHub Actions에서 돌아 라벨링된 한국어 페이지 10장의 점수 구간과 "디자이너 페이지 최저점 > 슬롭 페이지 최고점" 분리를 확인합니다. 실패하면 빌드가 빨개집니다.The checks are code, and CI enforces them. 22 tell detectors live in scripts/anti-ai-eval.js; five regression test files run on every push in GitHub Actions, pinning the score bands of ten labeled Korean pages and the "lowest designer page beats the highest slop page" separation. A failure reds the build.
  • Figma·Photoshop·호스팅을 대체하지 않습니다. 코드·SVG·토큰·스펙·카피를 만들어 그 도구들에 공급합니다. 여전히 그쪽으로 배포합니다.It doesn't replace Figma, Photoshop, or your host. It feeds them code/SVG/tokens/specs/copy; you still ship through those.
  • API 키 없이 "설치 0"이 아닙니다. 기획·디자인·코드·검수·토큰·카피는 추가 설치 없이 돌지만, 스크린샷/캡처 루프는 일회성 /design-setup(브라우저)이, 실제 이미지 생성은 OPENAI/GEMINI 키 또는 ChatGPT 로그인이 필요합니다."Zero setup" isn't universal. Plan/design/code/review/tokens/copy run with no extra install, but the screenshot/capture loop needs a one-time /design-setup (a browser), and real image generation needs an OPENAI/GEMINI key or a ChatGPT login.
  • lo-fi 컴프는 완성 아트가 아닙니다. 키가 없으면 이미지 슬롯을 의도된 온브랜드 SVG 자리표시로 채웁니다. 운영자가 실제 아트로 교체해야 합니다. 그리고 법적 검토·표시광고 적합성은 운영자 몫입니다.Lo-fi comps are placeholders, not finished art. Without a key, image slots render as intentional on-brand SVG comps the operator replaces. And compliance / claim-substantiation is left to you.

자주 묻는 질문Frequently asked questions

API 키가 꼭 필요한가요?Do I need an API key?

아니요. 기획·디자인·코드·검수·토큰·카피는 키 없이 동작합니다. 실제 사진/일러스트 생성만 OpenAI/Gemini 키 또는 무료 ChatGPT 로그인이 필요하고, 없으면 온브랜드 SVG 컴프로 대체합니다.No. Planning, design, code, review, tokens, and copy all work without one. Only real photo/illustration generation needs an OpenAI/Gemini key (or a free ChatGPT login); without it you get on-brand SVG comps.

설치하면 추가로 깔 게 없나요?Is there anything extra to install?

디자인 가이드와 토큰·린트 게이트, 빌트인 스크립트는 추가 npm 패키지 없이 됩니다. 스크린샷/접근성 게이트만 한 번 /design-setup 으로 브라우저(Patchright + Chromium)를 깔면 됩니다.The design guidance, the token/lint gates, and the built-in scripts need no extra npm packages. Only the screenshot/accessibility gate needs a one-time browser install (Patchright + Chromium) via /design-setup.

한국어 페이지 전용인가요?Is it only for Korean pages?

아니요. 한국형 상세페이지/와디즈 구조에 강하지만, 서구권 랜딩·앱 UI·브랜드도 같은 기준으로 만듭니다. 카피 린트는 한국어 로케일도 지원합니다.No. It's strong on Korean 상세페이지/Wadiz structure, but it builds western landing/app UI/brand to the same bar, and the copy lint supports a Korean locale too.

Figma를 대체하나요?Does it replace Figma?

아니요. 대체하지 않고 공급합니다. 코드·SVG·토큰·스펙·다이어그램·카피를 만들어 Figma·호스팅·이메일 발송 같은 다운스트림 도구에 넘깁니다. 설치형 디자인 스킬/패키지를 대체하는 거지, GUI 캔버스를 대체하는 게 아닙니다.No. It feeds them, it doesn't become them. It produces the code/SVG/tokens/specs/diagrams/copy that Figma, your host, and your ESP consume. It replaces other installable design skills/packages, not the GUI canvas.
