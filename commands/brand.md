# /brand — build a coherent brand identity

Brief: **$ARGUMENTS**

Invoke the **brand-identity** skill and run its loop — strategy before visuals, never a mood board with no reasoning.

1. **Strategy** — positioning (category, target, point of difference, reason to believe), audience & JTBD, a primary + secondary archetype, brand values, a one-line brand idea.
2. **Verbal identity** — voice & tone (3–4 traits with do/don't), messaging hierarchy (one-liner → elevator → boilerplate), naming/tagline direction, lexicon.
3. **Visual identity** — direct the logo (logotype vs mark, construction, clear space, misuse), a color system grounded in the strategy (OKLCH ramps; reject the AI-purple default), a type system (display + body; banned fonts excluded), imagery/illustration art direction, iconography, a motion signature.
4. **System & apply** — express the result as design tokens (hand to **design-system**) so every page/app built with **detail-page**, **ui-design**, and **frontend-build** comes out on-brand from one source. Produce the brand guidelines outline.

**Verify** brand coherence: does every touchpoint read as the SAME brand, and does it hold up beside a real competitor? If a logo image is wanted, generate it via the detail-page asset-generation flow. Deliver the system, then the single highest-leverage next step.
